# Язык проекта

Все ответы ассистента, комментарии и документация должны быть **на русском языке**.

Правила:

- ответы ассистента → всегда **на русском**
- документация проекта → **на русском**
- комментарии в коде → желательно на русском
- commit-сообщения → допустимы на английском или русском

Технические термины (API, event, instance, webhook и т.п.) могут использоваться на английском.

# Правила работы с Git

## Изменения только через Pull Request

Нельзя вносить изменения напрямую в основную ветку.

Процесс работы:

1. создать новую ветку
2. внести изменения
3. создать Pull Request
4. согласовать изменения
5. после согласования выполнить merge

Пример структуры веток:

main
 └ feature/название-задачи

---

## Обязательное согласование изменений

Перед изменением необходимо согласовать:

- архитектуру системы
- модель данных
- бизнес-логику
- интеграции

Согласование выполняется через обсуждение.

# Правила для Cursor-агента

При работе с проектом необходимо:

1. всегда отвечать **на русском языке**
2. не изменять архитектуру без согласования
3. не изменять модель данных без согласования
4. обновлять документацию при изменениях
5. вносить изменения **только через Pull Request**

Документы в папке **docs** являются источником истины для архитектуры и логики системы.
