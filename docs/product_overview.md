# Product Overview

## Проект: ассистент для терапевтов

## Краткое описание

Система автоматизирует административные процессы вокруг терапевтических встреч.

Основная задача:
- снизить количество ручных действий терапевта по напоминаниям об оплате
- упростить проверку статусов оплат встреч

Интеграции продукта:
- Google Calendar (источник расписания)
- Telegram (канал взаимодействия)
- Supabase (хранение данных системы)
- Make (оркестрация процессов)

---

# Проблема

Терапевты ведут расписание в Google Calendar, но выполняют административные задачи вручную:
- напоминают клиентам об оплате
- проверяют оплату
- сверяют статусы встреч
- ежедневно просматривают календарь

Эти действия повторяются и создают постоянную административную нагрузку.

---

# Цель продукта

Создать минимальную автоматизацию, которая:
- синхронизирует встречи из календаря
- отправляет напоминания об оплате
- фиксирует факт оплаты
- показывает терапевту статус встреч

Ключевая цель: снизить административную нагрузку терапевта.

---

# Целевые пользователи

Основной пользователь: частный терапевт, который:
- ведет расписание в Google Calendar
- общается с клиентами через Telegram
- получает оплату напрямую от клиентов

Решение ориентировано на индивидуальных специалистов, а не на клиники.

---

# Основные функции

## Синхронизация встреч

Система синхронизирует события из Google Calendar.

Календарь является источником истины для расписания.

Система не создает и не изменяет события календаря.

## Напоминания об оплате

Система отправляет клиенту reminder об оплате по глобальным настройкам терапевта (задаются при стартовой настройке и в меню бота):
- `payment_reminder_timing`: `before` или `after`
- `payment_reminder_offset_minutes`: смещение в минутах

Для каждого клиента терапевт в боте включает или выключает напоминания:
- `send_payment_reminders`

## Подтверждение оплаты

Клиент подтверждает оплату кнопкой в Telegram.

После подтверждения:
- `calendar_event_instances.is_paid = true`
- `calendar_event_instances.payment_confirmed_at = now()`

## Daily digest для терапевта

Каждый день в заданное время (по умолчанию 09:00 в таймзоне терапевта, настраивается в боте) терапевт получает список встреч со статусами оплат.

---

# Границы продукта

В первую версию не входят:
- бронирование встреч
- перенос встреч клиентом
- платежная система
- CRM клиентов
- сложная аналитика
- управление несколькими терапевтами

Система автоматизирует только административные действия вокруг встреч.

---

# Ожидаемый результат

После внедрения терапевт:
- тратит меньше времени на административные операции
- быстрее видит статусы оплат по встречам

Система снижает количество ручных действий без изменения базового рабочего процесса терапевта.
